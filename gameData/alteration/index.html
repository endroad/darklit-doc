    alteration.new

    (void)alteration.new:(string)alter duration:(float)duration priority:(float)priority

    Most of the time Alters are simple enough to just use the Inflict Message and then let it go but then some alters require more complexity than others, this message creates a new alter, based on the name and push it as the context for this entity, you may then proceed in parametter setters using the provided SET messages, much like the Attack context. Unlike the Attack context however, properties of Alters context are solely based on the Ins and Outs of the alter script. Exemple : Poison has a "in int damages", thus the parameter int damages will be available for editing. When you are done with editing, use the message Push to inflict the alter, context will not be available after that.

    Arguments

    string alter: name of the alter, crucial

    float duration: how long this alter will stay when inflicted

    float priority: how "powerful" this alter is, meaning which version will be used in case of multiple inflict of the same alter

    Returns

    N/A

    Example

    1
2
    // This example is automatically generated: don't trust values
[this alteration.new:"helloWorld" duration:2.9 priority:2.9]
